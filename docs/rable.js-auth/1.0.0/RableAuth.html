    Parameters:
    Name Type Description

    The input / configuration of the Library.

    Properties
    Name Type Default Description
    callbacks object

    [REQUIRED] Callbacks of the Library.

    Properties
    Name Type Description
    obtainUser function

    [REQUIRED] Obtain the user based on their identifier. callback should return hashed password (bcrypt) and a systemIdentifier stored in a value identical to config option systemIdentifier. (identifier could be a username or email, not managed by the library. systemIdentifier could be a id, uuid or similar specific to the targeted user)
    onLogin function

    Fired when somebody successfully logs in. systemIdentifier will be provided through the first parameter.
    apiController RableAPI

    [REQUIRED] rable.js-api instance. already configured with the rable.js instance.
    rableInstance Rable

    [REQUIRED] rable.js instance.
    systemIdentifier string

    [REQUIRED] systemIdentifier name for the users.
    userTokenSecret string

    random string which will be used to sign and verify the tokens. All previously made tokens will become invalid once changed. If none provided, the Library will generate a random secret.
    cookieName string

    Name of the cookie where the token should be stored, if any. (not required, if no cookieName is defined, the token will be retured to the requestmaker)
    cookieOpts object

    If a cookie will be set, the options for it. path will be set to '/' by default. refer to rable.js documentation for valid options.
    returnToken boolean false

    In case a cookieName was set, this will state if the token should be returned back to the user.
    api string auth

    Name of the API. (An existing API will not be overwritten, only the routes required for the extention to work)
    tokenExpiration string 1h

    How long a token with expiration should last.
    allowNoExpiration string false

    States if a token without an expiration is allowed to be provided.
    autoRefresh boolean true

    In case the token is transported in a cookie, states if the cookie should automatically be updated with the new token. (req.newToken)
    Properties
    Name Type Description
    version string

    The current version of the Library.
    config object

    The configuration of the Library that was provided on initialization. Some variables were moved to private variables to make them unchangable.
